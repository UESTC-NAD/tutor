# 部署到 GitHub Pages

只有第一次麻烦，之后每次更新就是三条命令。

## 一、注册并创建仓库（只做一次）

1. 打开 https://github.com/ 注册账号（如果还没有）
2. 登录后点右上角 **+** → **New repository**
3. 填写：
   - **Repository name**: `tutor`（或你喜欢的名字，会出现在网址里）
   - **Public**（必须公开，私有仓库的 Pages 需要付费账户）
   - **不要**勾选 "Add a README file"
4. 点 **Create repository**

创建完成后页面上会显示仓库地址，形如：

```
https://github.com/你的用户名/tutor.git
```

## 二、把本地文件推上去（只做一次）

在 `E:\OneDrive\杂项\tutor-site` 目录下依次运行（把 `你的用户名` 换成实际的）：

```bash
git remote add origin https://github.com/你的用户名/tutor.git
```

```bash
git branch -M main
```

```bash
git push -u origin main
```

第一次 push 会弹出登录窗口，用浏览器授权 GitHub 账号即可。

## 三、打开 Pages 开关（只做一次）

1. 进入仓库页面 → 顶部 **Settings**
2. 左侧菜单找到 **Pages**
3. **Source** 选 `Deploy from a branch`
4. **Branch** 选 `main`，文件夹选 `/ (root)`，点 **Save**
5. 等 1–2 分钟，页面顶部会出现网址：

```
https://你的用户名.github.io/tutor/
```

**这个网址就是给弟弟的 iPad 用的。** 用 Safari 打开，加到主屏幕（分享按钮 → 添加到主屏幕），
之后点图标就像 App 一样直接进。

## 四、以后每次加新教辅

1. 把新的 HTML 丢进 `lessons/` 文件夹
2. 在 `tutor-site` 目录运行：

```bash
node build.mjs
```

```bash
git add -A
```

```bash
git commit -m "新增教辅"
```

```bash
git push
```

等 1–2 分钟，iPad 上刷新页面就能看到了。

## 常见问题

**Q：推送时提示 `remote origin already exists`**
说明之前已经加过了，直接跳到 `git push -u origin main`。
想换地址就先运行 `git remote set-url origin 新地址`。

**Q：iPad 上打开是旧内容**
Safari 缓存。下拉刷新，或者在地址栏网址后面加个 `?v=2` 强制刷新。

**Q：公式还是显示成 `\( ... \)` 代码**
说明那份教辅的 MathJax 引用地址不对。检查它 `<head>` 里是不是
`<script defer src="../assets/mathjax-tex-svg.js"></script>`，
详见 README.md 的「关于公式渲染」一节。

**Q：不想让别人看到内容**
仓库必须公开才能免费用 Pages，但网址不会被公开索引，别人猜不到就访问不到。
如果确实需要私密，可以改用 Cloudflare Pages（免费版支持给站点加访问密码）。
