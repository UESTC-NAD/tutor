# 部署到 GitHub Pages

GitHub 账号：**UESTC-NAD**

本地该配的都配好了（仓库已初始化、远程地址已填、首次提交已完成）。
剩下三件事必须你自己在网页上操作，因为要登录你的账号。

## 一、在 GitHub 上建仓库（只做一次）

1. 打开 https://github.com/new
2. 填写：
   - **Repository name**: `tutor`
   - **Public**（必须公开，私有仓库开 Pages 要付费）
   - **不要**勾选 "Add a README file"、"Add .gitignore"、"Choose a license"
     （本地已经有这些文件了，勾了会冲突）
3. 点 **Create repository**

建完先别管页面上那一堆命令，直接看下一步。

## 二、推送（只做一次）

在 `E:\OneDrive\杂项\tutor-site` 目录下运行：

```bash
git push -u origin main
```

第一次会弹出登录窗口，用浏览器授权 UESTC-NAD 账号即可。

> 远程地址 `https://github.com/UESTC-NAD/tutor.git` 已经配好了，不用再 `git remote add`。

## 三、打开 Pages 开关（只做一次）

1. 进入 https://github.com/UESTC-NAD/tutor
2. 顶部 **Settings** → 左侧菜单 **Pages**
3. **Source** 选 `Deploy from a branch`
4. **Branch** 选 `main`，右边文件夹选 `/ (root)`，点 **Save**
5. 等 1–2 分钟刷新页面，顶部会出现网址：

```
https://uestc-nad.github.io/tutor/
```

**注意网址里的用户名是全小写的** `uestc-nad`，这是 GitHub 的规则，不是写错了。
大写的 `UESTC-NAD` 也能访问，会自动跳转到小写。

**这个网址就是给弟弟的。** 用 Safari 打开 → 分享按钮 → 添加到主屏幕。

## 常见问题

**Q：想换个仓库名怎么办**
建仓库时用别的名字，然后运行
`git remote set-url origin https://github.com/UESTC-NAD/新名字.git`，
网址也跟着变成 `https://uestc-nad.github.io/新名字/`。

**Q：能不能让网址短一点**
可以。把仓库名改成 `UESTC-NAD.github.io`，网址就变成 `https://uestc-nad.github.io/`，不带后面的 `/tutor`。
代价是每个账号只能有一个这种仓库，以后想做个人主页就占用了。

**Q：推送时提示 `Repository not found`**
仓库还没在网页上建，或者名字不一致。回到第一步。

**Q：推送时提示 `remote origin already exists`**
说明已经配过了（本来就配好了），直接 `git push -u origin main`。

**Q：Pages 设置里找不到 Branch 选项**
仓库还是空的。先完成第二步的 push，有内容之后选项才会出现。

**Q：iPad 上打开是旧内容**
Safari 缓存。下拉刷新，或在网址后面加 `?v=2` 强制刷新。

**Q：公式还是显示成 `\( ... \)` 代码**
那份教辅的 MathJax 引用地址不对。检查它 `<head>` 里是不是
`<script defer src="../assets/mathjax-tex-svg.js"></script>`，
详见 README.md 的「关于公式渲染」一节。

**Q：不想让别人看到内容**
仓库必须公开才能免费用 Pages，但网址不会被搜索引擎收录，别人猜不到就访问不到。
如果确实需要密码保护，可以改用 Cloudflare Pages（免费版支持给站点加访问密码）。
